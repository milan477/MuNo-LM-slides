Music understanding is hard for ALMs

What is played

Score: melody, harmony, phrasing, dynamics, form, voicing, articulation — the composer's intent.

How it is played

Performance: precise onset/offset times, velocities, rubato, expressive shaping — the performer's interpretation.

Existing ALM training pipelines rely on LLM-synthesized captions and metadata that are coarse and weakly time-grounded — limiting fine-grained musical reasoning.

→

No existing representation unifies both score semantics and performance timing in an LLM-readable format.

Pipeline — from raw files to training-ready QA

①

Parse Score

MusicXML → note graph: pitch, duration, voice, dynamics, slurs, articulation per measure

②

Align Performance

MIDI → DTW on chroma features → each score note gets onset / offset timestamps + MIDI velocity

③

Render MuNo-LM

Merge score graph + timing → hierarchical text: BAR / PART / VOICE / NOTE structure

④

LLM Caption

Feed MuNo-LM to GPT-5.5 / Gemini → time-grounded auditory description with timestamp anchors

⑤

Extract QA

Claude Sonnet reads description → short-form Q&A pairs with verifiable exact-match answers

Verbosity variants

The MuNo-LM renderer produces four variants — Verbose (all fields), Compact (omit redundant defaults), Minimal (pitch + timestamp only), Beats (bar-relative beat positions instead of wall-clock time). Each tested independently in SUBench.

Scale

236 scores × multiple recordings each. Captioning cost: ~$0.04/piece at GPT-5.5 pricing. Full (n)ASAP caption run: under $50 total. Fully automated — no human annotation beyond SUBench QA validation.
